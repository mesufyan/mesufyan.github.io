Master's thesis · current

Automating Quality Assurance in Radiotherapy Clinical Trials

Role: Graduate ResearcherWhere: LPC Caen & Centre François BaclesseSupervisor: Prof. Juliette ThariatStack: Python, PyTorch, Pydicom

Radiotherapy QA tells you a plan failed, not why. I built a geometric descriptor of anatomical complexity so QA can tell a badly-planned case apart from an intrinsically hard one — the foundation of my PhD.

The problem

In radiotherapy clinical trials, every treatment plan is checked by experts before delivery — quality assurance (QA). Today that check is largely manual, expert-dependent, and returns a blunt verdict: pass or fail. When a plan fails, the verdict doesn’t say why.

That ambiguity matters. A plan can fail because it was planned badly, or because the patient’s anatomy is intrinsically so complex that no plan could fully spare the organs at risk. Treating both the same way makes QA inconsistent across centres and hard to standardise or scale.

My approach — Quantify, Integrate, Automate

The thesis introduces a way to measure how hard a case actually is, integrate that measure into the QA verdict, and ultimately learn it automatically.

PHASE I

Quantify

Define Tumour Topology Complexity (TTC) per organ and per patient.

PHASE II

Integrate

Redefine QA into four meaningful classes using the descriptor.

PHASE III

Automate

Predict the new classes with multimodal attention-based learning.

What I have done so far

A geometric descriptor: the Geometric Volume Coefficient (GVC)

Starting from each organ’s differential distance-to-target histogram (dDTH) — how much organ volume sits at each distance from the target — I split the organ into three zones: the overlapped volume (intrinsic complexity), the near-compression volume that can’t be spared because of machine dose-gradient limits, and the safe volume far enough to spare. These combine into a normalised GVC on a 0–1 scale, where a higher value means a harder organ.

From organ to patient

Organ-level GVCs are aggregated into a single patient-level score using a quadratic, self-weighted formulation (GVC²×w): high-complexity organs dominate without low-complexity ones being ignored, and each organ also carries a clinical-priority weight drawn from QUANTEC/RTOG-type standards rather than chosen arbitrarily. The result is standardised and reproducible across centres.

A fairer QA verdict

With the descriptor in hand, a failed plan can be split apart. If targets and organs aren’t both satisfied but the TTC is low, that’s a genuinely suboptimal plan. If the TTC is high, it’s a complexity-limited case — theoretically a fail, but clinically the best achievable. The proposed QA therefore reports four classes instead of two:

  • True — all goals met.
  • False — no goals met.
  • Suboptimal — one goal missed, low complexity (planning can improve).
  • TTC-limited — compromise forced by high anatomical complexity.

Where this is heading

The next step is automation: training models to predict these four classes directly from data. I plan to start with attention-based Multiple Instance Learning on tabular clinical and DVH data, then bring in patient DICOM (CT, RTDOSE), and finally fuse both streams in a multimodal framework on the REACH multi-institutional database. A first practical check is already in view — using organ-level GVCs as input features for toxicity prediction, since higher complexity should track higher toxicity risk.

Status: descriptor formulated and validated on real patient data; integration logic defined; multimodal automation in progress. Code repository going public on GitHub soon.
